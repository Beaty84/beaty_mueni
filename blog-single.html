Blogging With Jekyll

  • Beaty

What is Jekyll?

Jekyll is a simple, blog aware static site generator. It is designed for building minimal to static blogs to be hosted on github pages. It takes text written in your favorite markup language and uses layout to create a static website. You can tweak the site's look and feel, URLs, the data displayed and more. Jekyll is the engine behind gh-pages which you can use to host sites right from your github repository.

Why Use Jekyll?

  • Simplicity- Jekyll strips everything down to the bare minimum,eliminating a lot of complexity.
  • No database- This speeds up loading since no database calls are made when a site is loaded.
  • No CMS- Simply write all your content in markdown and Jekyll will run it through templates to generate your static website
  • Minimal - Your jekyll website will include absolutely no functionality or features that you aren't using.
  • Design control- create your own theme or customize an uncomplicated base theme.
  • Convenient hosting-Github pages will build and host your jekyll website at no charge while simultaneously handling version control.

Getting Started With Jekyll

Using Jekyll with GH-Pages to create a blog

You can Install jekyll locally via command line but the easiest way to get started with jekyll is by:
  1. Forking a starting point
Steps
Create a repository on Github
navigate to Jekyll themes on your browser. This will open the many Jekyll themes, and from this, you can choose one which you want your blog to look like. For me, i love using beautifull_Jekyll.
Navigate back to your repo on Github and search the theme you've chosen. Click on one of the repos on the search results.
To the top right corner of the repo, hit the fork button to fork a copy of the blog theme to your github account.

  1. Hosting on github pages
Click on settings button in your new forked repo and change the repo name to yourusername.github.io

Progress, not Perfection

3. Customizing You can change your site's name, description, avatar, and other options by editing the _config.yml file and committing the changes. Making a change to the _config.yml file forces gh-pages to rebuild your website with Jekyll. To customize and theme a jekyll website, you just need to place the files that you want to be built in the gh-pages branch of your repo and then github pages will process it with jekyll

4. Publishing your blog Blogs are made up of posts, and jekyll has the concept built in its default structure Edit /_posts/ deleting the placeholder content and entering your own. Change the file name to include today's date and the title of your post in the format(year-month-day-title.md.) To create a blog post , simply hit the “+” icon in /_posts/ and include the front-matter block so that the files get processed by jekyll. Front matter tells gh-pages site whether to format the content below the front matter as a page or blog post Example ----- Layout: post Title: … -----
